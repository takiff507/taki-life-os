# Contributing

Thanks for improving Taki Life OS. The project is intentionally simple: static HTML, CSS, and vanilla JavaScript.

## Development Flow

1. Create a focused branch for one change.
2. Keep changes small enough to review quickly.
3. Test the affected workflow manually.
4. Open a pull request with a short summary and test notes.

## Good Contributions

- Bug fixes with clear reproduction steps.
- Small usability improvements.
- Documentation that helps users understand local-first data.
- Accessibility and responsive layout fixes.

## Style

- Prefer readable vanilla JavaScript over extra dependencies.
- Keep UI text short and practical.
- Avoid adding build tools unless the benefit is clear.
- Keep data import/export backward compatible when possible.

## Pull Request Notes

Include:

- What changed
- Why it changed
- How it was tested
