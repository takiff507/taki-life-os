# Release Checklist

Use this checklist before announcing or sharing a new version.

## Code

- Confirm there are no console errors on first load.
- Confirm all navigation tabs open correctly.
- Confirm data saves after refresh.
- Confirm export/import works with a fresh backup file.

## Content

- Update README feature notes if behavior changed.
- Update docs when data format or workflows change.
- Add screenshots only after the UI is stable.

## GitHub Pages

- Confirm Pages is enabled for the repository.
- Wait for the deployment to finish.
- Open the live URL in a private/incognito window.
- Test the dashboard on mobile width.

## After Release

- Create an issue for any follow-up bugs.
- Keep small improvements in separate pull requests.
